# x-cisco-metadata Extension

A Cisco DevNet specification extension for MCP Description documents that captures runtime observations, dump provenance, CORS browser-readiness, and connection diagnostics.

## Overview

The `x-cisco-metadata` extension enriches MCP Description documents with metadata discovered during runtime capability extraction using the [mcpcontract CLI](https://github.com/cisco-open/mcptoolkit-contract).

This extension follows the MCP Description [specification extension mechanism](../../sections/13-specification-extensions.md).

## Properties

### Top-Level

| Property | Type | Description |
|----------|------|-------------|
| `version` | string | Extension payload version (`"0.2.0"`) |
| `dump` | object | Dump-derived metadata payload |

### dump

| Property | Type | Description |
|----------|------|-------------|
| `toolName` | string | Tool used to create the dump (e.g., `"mcpcontract"`) |
| `toolVersion` | string | Version of the dump tool |
| `createdAt` | string (date-time) | When the extraction was performed |
| `serverConfig` | object | Connection configuration used during extraction |
| `runtimeObservations` | object | Runtime discoveries from MCP initialization |
| `cors` | object | CORS support detection for browser-based clients |
| `paginationDetection` | object | Pagination behavior per capability |
| `clientCapabilities` | object | Client capabilities sent during initialization |
| `splitOperation` | object | Split operation metadata (when generated from a split) |
| `enrichment` | object | Additional content like response examples |

### runtimeObservations

| Property | Type | Description |
|----------|------|-------------|
| `mcpProtocolUsed` | string | Protocol version used during communication |
| `sessionIdSupported` | boolean | Whether the server returned a session ID |
| `sessionIdHeader` | string | Exact session header name with casing |
| `pingSupported` | boolean | Whether the server responded to ping |
| `pingLatencyMs` | number | Round-trip ping latency in milliseconds |
| `instructions` | string | Server instructions from InitializeResult |

### cors

| Property | Type | Description |
|----------|------|-------------|
| `browserReady` | boolean/null | Overall browser readiness assessment |
| `responseHeaders` | object | CORS headers from initialize response |
| `preflight` | object | Results from OPTIONS preflight request |

## Example Usage

```json
{
  "mcpdesc": "0.7.0",
  "info": { "name": "my-server", "version": "1.0.0" },
  "transports": [{ "type": "streamable-http", "url": "https://example.com/mcp" }],
  "tools": [{ "name": "my_tool", "description": "A tool" }],

  "x-cisco-metadata": {
    "version": "0.2.0",
    "dump": {
      "toolName": "mcpcontract",
      "toolVersion": "0.8.0",
      "createdAt": "2026-03-15T14:30:00Z",
      "runtimeObservations": {
        "mcpProtocolUsed": "2025-06-18",
        "sessionIdSupported": true,
        "sessionIdHeader": "Mcp-Session-Id",
        "pingSupported": true,
        "pingLatencyMs": 42
      },
      "cors": {
        "browserReady": true,
        "responseHeaders": {
          "accessControlAllowOrigin": "*"
        }
      }
    }
  }
}
```

## Generating This Extension

The `x-cisco-metadata` extension is typically auto-generated by the `mcpcontract` toolkit:

```bash
# Extract capabilities (produces a dump with runtime observations)
mcpcontract dump --server-name my-server --url https://example.com/mcp --transport streamable-http --output dump.json

# Generate manifest (converts dump to MCP Description with x-cisco-metadata)
mcpcontract manifest --dump dump.json --info server-info.json --output manifest.mcpdesc.json
```
