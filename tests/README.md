# Test Suite Organization

This folder contains all test-related files for the `mcpcontract` project.

## Structure

```
tests/
├── unit/                  # Jest unit tests
├── integration/           # Jest integration tests (end-to-end)
├── generators/            # Catalog → test generators
├── fixtures/              # Test data
│   ├── catalogs/          # Catalog validator fixtures
│   ├── dumps/             # Capability dump fixtures (used by unit tests)
│   └── split/             # Split-config fixtures
├── check-doc-links.sh     # Documentation link validator
└── run-manual-tests.sh    # Manual CLI smoke tests (requires running server)
```

## Running Tests

```bash
# Build first
npm run build

# Full Jest suite (unit + integration)
npm test

# Watch mode
npm run test:watch

# Coverage
npm run test:coverage

# Auto-generate tests from catalog YAML
npm run test:generate

# Manual smoke tests (requires a running MCP server at http://localhost:3000/mcp)
npm run test:manual

# Documentation link checker
npm run test:links

# Pre-release: link check + build + full test suite
npm run prerelease
```

## Test Inventory

- **Unit** (`tests/unit/`) — rules engine, splitter, completion script, HTTP headers, etc.
- **Integration** (`tests/integration/`) — full workflows (dump → manifest → validate → diff → changelog)
- **Auto-generated** (`tests/unit/catalog-generated.test.ts`) — one test per `rules/catalog/**/*.yaml` example, regenerated by `tests/generators/catalog-test-generator.ts`

Current count: 73+ tests (47 auto-generated + 15 unit + 11 integration).

## Adding New Fixtures

Use a generic placeholder name (e.g., `sample-server`):

```bash
# 1. Create a dump
mcpcontract dump \
  --server-name sample-server \
  --transport streamable-http \
  --url http://localhost:3000/mcp \
  --output tests/fixtures/dumps/sample-server-dump.json --pretty

# 2. Create server info at tests/fixtures/info/sample-server-info.json
#    (schema: schemas/manifest-info/1.0.0.json)

# 3. Generate manifest
mcpcontract manifest \
  --mcpdesc tests/fixtures/dumps/sample-server-dump.json \
  --info tests/fixtures/info/sample-server-info.json \
  --output tests/fixtures/manifests/sample-server-manifest.json --pretty
```

## Naming Conventions

- Config files: `<server-name>-config.json`
- Dump files:   `<server-name>-dump.json`
- Info files:   `<server-name>-info.json`
- Manifests:    `<server-name>-manifest.json` or `<server-name>-server.json`
